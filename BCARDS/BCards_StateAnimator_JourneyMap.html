② StateAnimator — módulo de controle por estado
Arquitetura: o StateAnimator é um módulo passivo que observa o ENGINE.state.phase e dispara callbacks, classes CSS e hooks de animação mapeados por estado. Não substitui RENDERER nem UI — fica numa camada acima, como um roteador de efeitos visuais. Integra com o GPC via GPC.t() para respeitar a velocidade atual.
Onde inserir no HTML: adicionar como <script id="state-animator"> logo após o bloco ctx-icon-system e antes do battle-fx-module. Expor via window.STATE_ANIM para debugging. 
/* ════════════════════════════════════════════════════════════════
   STATE ANIMATOR — Elemental Clash v29
   ════════════════════════════════════════════════════════════════
   Módulo de roteamento de efeitos visuais por estado de jogo.

   USO:
     STATE_ANIM.enter('p1select')   // dispara manualmente (para testes)
     STATE_ANIM.onEnter('reveal', fn)  // registra hook externo
     STATE_ANIM.currentPhase()      // lê fase ativa
   ════════════════════════════════════════════════════════════════ */
'use strict';
const STATE_ANIM = (() => {

  // ── Fase atual rastreada internamente ───────────────────────────
  let _phase = null;

  // ── Mapa de hooks externos (plugins / debugging / future use) ───
  const _hooks = {}; // { phase: [fn, fn, ...] }

  // ── MAPA DE ESTADOS ─────────────────────────────────────────────
  // Cada entrada define o que acontece quando o jogo ENTRA naquela fase.
  // onEnter: chamado imediatamente ao entrar
  // classes: { selector: [add], [remove] } — aplicadas no DOM
  // Todos os timers devem usar GPC.t(baseMs) para respeitar a velocidade.

  const STATE_MAP = {

    // ──────────────────────────────────────────────────────────────
    // p1select — P1 ativo, escolhendo carta
    // ──────────────────────────────────────────────────────────────
    'p1select': {
      label: 'P1 Selecionando',
      onEnter() {
        // Zona ênfase já aplicada pelo RENDERER.renderPhase → applyZoneEmphasis
        // Ctx-icon já aplicado pelo updateCtxIcon('p1select')
        // Este hook é reservado para efeitos adicionais futuros:
        _dispatchHooks('p1select');
      }
    },

    // ──────────────────────────────────────────────────────────────
    // p2select — P2 ativo, P1 já confirmou
    // ──────────────────────────────────────────────────────────────
    'p2select': {
      label: 'P2 Selecionando',
      onEnter() {
        _dispatchHooks('p2select');
      }
    },

    // ──────────────────────────────────────────────────────────────
    // pre-reveal — ambos confirmaram, aguardando clique em REVELAR
    // (estado visual: overlay reveal-focus visível com btn-reveal)
    // ──────────────────────────────────────────────────────────────
    'pre-reveal': {
      label: 'Aguardando Revelação',
      onEnter() {
        // reveal-focus.show já aplicado pelo RENDERER.renderPhase
        // Este hook permite adicionar sons, hápticos, efeitos de antecipação
        _dispatchHooks('pre-reveal');
      }
    },

    // ──────────────────────────────────────────────────────────────
    // reveal — decks virando, cartas expostas
    // ──────────────────────────────────────────────────────────────
    'reveal': {
      label: 'Revelação',
      onEnter() {
        _dispatchHooks('reveal');
      }
    },

    // ──────────────────────────────────────────────────────────────
    // fight-impact — pico da animação de combate
    // T=0 entrada do punch · T=480ms soco · T=580ms empurrão
    // ──────────────────────────────────────────────────────────────
    'fight-impact': {
      label: 'Impacto',
      onEnter(result) {
        // result = objeto de resolveRound() passado pelo chamador
        if (!result) { _dispatchHooks('fight-impact'); return; }
        // Transição automática para fight-result após duração do impacto
        setTimeout(() => {
          STATE_ANIM.enter('fight-result', result);
        }, GPC.t(1050)); // punchWinner loop começa em T+1050ms
        _dispatchHooks('fight-impact', result);
      }
    },

    // ──────────────────────────────────────────────────────────────
    // fight-result — vencedor em loop, resultado visível, HP atualizado
    // ──────────────────────────────────────────────────────────────
    'fight-result': {
      label: 'Resultado do Combate',
      onEnter(result) {
        // Transição automática para round-end após apresentação do resultado
        setTimeout(() => {
          STATE_ANIM.enter('round-end', result);
        }, GPC.t(750)); // 1800ms total - 1050ms = 750ms entre fight-result e toast
        _dispatchHooks('fight-result', result);
      }
    },

    // ──────────────────────────────────────────────────────────────
    // round-end — toast visível, aguarda próxima rodada ou game over
    // ──────────────────────────────────────────────────────────────
    'round-end': {
      label: 'Fim de Rodada',
      onEnter(result) {
        _dispatchHooks('round-end', result);
      }
    },

    // ──────────────────────────────────────────────────────────────
    // gameover — tela de game over visível
    // ──────────────────────────────────────────────────────────────
    'gameover': {
      label: 'Game Over',
      onEnter(result) {
        _dispatchHooks('gameover', result);
      }
    },

    // ──────────────────────────────────────────────────────────────
    // done — estado de repouso pós-round (antes de nextRound)
    // ──────────────────────────────────────────────────────────────
    'done': {
      label: 'Aguardando Próxima Rodada',
      onEnter() {
        _dispatchHooks('done');
      }
    },
  };

  // ── DISPATCHER ──────────────────────────────────────────────────
  function _dispatchHooks(phase, payload) {
    const fns = _hooks[phase];
    if (!fns) return;
    fns.forEach(fn => {
      try { fn(payload); }
      catch(e) { console.warn(`[STATE_ANIM] Hook error on "${phase}":`, e); }
    });
  }

  // ── API PÚBLICA ─────────────────────────────────────────────────

  /**
   * enter(phase, payload?)
   * Transiciona para um estado e executa seu onEnter.
   * payload é passado para hooks (ex: result de resolveRound).
   */
  function enter(phase, payload) {
    const def = STATE_MAP[phase];
    if (!def) {
      console.warn(`[STATE_ANIM] Estado desconhecido: "${phase}"`);
      return;
    }
    if (_phase === phase) return; // idempotente — não re-entra no mesmo estado
    _phase = phase;
    if (def.onEnter) def.onEnter(payload);
  }

  /**
   * onEnter(phase, fn)
   * Registra um hook chamado sempre que o estado é ativado.
   * Pode ser chamado múltiplas vezes para o mesmo estado.
   * Exemplo:
   *   STATE_ANIM.onEnter('fight-impact', result => playSound('clash'));
   */
  function onEnter(phase, fn) {
    if (!_hooks[phase]) _hooks[phase] = [];
    _hooks[phase].push(fn);
  }

  /** Retorna a fase atual rastreada. */
  function currentPhase() { return _phase; }

  /** Lista todos os estados disponíveis com seus labels. */
  function states() {
    return Object.entries(STATE_MAP).map(([id, def]) => ({
      id,
      label: def.label,
    }));
  }

  /** Reset — útil em restart() */
  function reset() { _phase = null; }

  return { enter, onEnter, currentPhase, states, reset };
})();
③ Guia de integração no BCards_v28.html
Abordagem: o StateAnimator é aditivo — não reescreve nada do engine existente. Cada ponto de integração é um call único adicionado ao final de funções já existentes.
1
Inserir o módulo no HTML após <script id="ctx-icon-system"> e antes de <script id="battle-fx-module">. O módulo usa GPC.t() que já existe — sem novas dependências.
2
Em renderPhase() do RENDERER, adicionar ao final de cada bloco condicional: 
if (G.phase === 'p1select') {
  applyZoneEmphasis('p1');
  STATE_ANIM.enter('p1select'); // ← ADICIONAR
} else if (G.phase === 'p2select') {
  applyZoneEmphasis('p2');
  STATE_ANIM.enter('p2select'); // ← ADICIONAR
} else if (G.phase === 'reveal') {
  STATE_ANIM.enter('pre-reveal'); // ← ADICIONAR (overlay visível = pre-reveal)
} else {
  resetZoneEmphasis();
}
3
Em onReveal() do UI layer, adicionar após BATTLE_FX.resolveResult(result): 
// T+500ms — battle FX dispara
setTimeout(() => {
  BATTLE_FX.resolveResult(result);
  showPunchIcon(result.outcome === 'tie' ? null : result.outcome);
  STATE_ANIM.enter('fight-impact', result); // ← ADICIONAR
}, GPC.t(500));
4
No callback do TOAST_CTRL.show(), o STATE_ANIM.enter('round-end') já é chamado automaticamente pela transição interna de fight-result → round-end. Nenhuma alteração necessária no código do toast.
5
Em onRestart(), adicionar STATE_ANIM.reset() antes de ENGINE.restart(): 
function onRestart() {
  _pprIconLock = false;
  STATE_ANIM.reset(); // ← ADICIONAR
  RENDERER.hideGameOver();
  // ... resto igual
}
6
Uso de hooks externos (exemplo de extensão futura sem tocar no core): 
// Som de impacto ao entrar em fight-impact

  if (result?.outcome !== 'tie') AudioEngine.play('clash');
});

// Vibração háptica em round-end
STATE_ANIM.onEnter('round-end', () => {
  navigator.vibrate?.([30, 50, 30]);
});

// Debug — loga cada transição de estado
STATE_ANIM.states().forEach((s) => {
  STATE_ANIM.onEnter(s.id, () => {
    console.log(`[STATE] → ${s.id} (${s.label})`);
  });
});
